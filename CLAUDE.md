## 1. Mission & guiding principle

**Clutch** is an **autonomous deadline-rescue agent** — an AI productivity companion
that proactively helps users *complete* tasks before deadlines are missed.

**The one rule that governs every feature:**
> **It doesn't remind — it does the work and shows me.**

If a feature only notifies/tracks, redesign it so the agent *takes an action*
(decompose, schedule, draft, generate) or cut it.

## 3. Product spec — the 4 hero capabilities

1. **Autonomous Triage & Rescue Plan** — user dumps commitments in plain language (or syncs
   Calendar); agent returns a *ranked rescue plan* (risk = deadline × importance × % remaining),
   not a flat list. It explains what it will *do* about each item.
2. **Calendar Surgery** — agent reads the user's **real Google Calendar**, finds free slots, and
   **creates work-block events** for each rescued task.
3. **Do-the-Work Actions** — agent produces real artifacts, not advice:
   essay/assignment → outline + first draft section; "ask for extension" → ready-to-send email
   draft; bill → parsed amount/date + one-tap action; interview → tailored prep questions.
4. **Proactive Risk Scan** — on app load (and optionally on a schedule / by voice), the agent
   runs **unprompted**: "3 things are about to slip — want me to handle them?"

**Mandatory UI element: the Agent Activity Feed** — a live, streaming trail of every tool the
agent calls (e.g. "Decomposed 'Assignment' → 4 steps · Found 3 free slots Wed–Fri · Created event
'Draft intro' Wed 6 PM · Drafted extension email"). This feed is the visible proof of agentic
depth and must be prominent.

## 4. Tech stack (pinned — do not swap without asking)

- **Framework:** Next.js 15 (App Router) + TypeScript.
- **Styling/motion:** Tailwind CSS + Framer Motion (cinematic, calm "command center" feel; mobile-friendly).
- **AI:** `@google/genai` SDK. **Model: `gemini-3.5-flash`** (stable, agentic). Optional heavy
  planning: `gemini-3.1-pro-preview`. Optional voice: `gemini-3.1-flash-live-preview` (Live API).
- **Agentic core:** **Gemini function calling** (tool loop). Verify exact SDK syntax against
  https://ai.google.dev/gemini-api/docs/function-calling before relying on it.
- **Auth + data:** Firebase Auth (Google provider) + Firestore.
- **Calendar:** Google Calendar API v3, via the user's OAuth token (Google Identity Services in the
  browser → token passed to the server route). Scope: `https://www.googleapis.com/auth/calendar.events`.
- **Deploy:** Docker → **Cloud Run** (`gcloud run deploy`). Next.js `output: 'standalone'`.

## 5. Architecture

```
Browser (Next.js client)                Next.js server (App Router route handlers)
─────────────────────────               ──────────────────────────────────────────
- Today / Rescue view          ──►      /api/agent   ← runs the Gemini tool-loop
- Agent Activity feed (live)            (GEMINI_API_KEY stays server-side)
- Google Sign-In (GIS) ──┐                 │ executes tools, returns plan + action log
  gets Calendar token    └──► token ──►    ├─ Calendar tools use the passed user token
- Firestore (tasks/state)                  └─ artifacts/email drafts generated by Gemini
```

**The agent loop (server, in `/api/agent`):**
`perceive → plan → act → observe → repeat` until the model returns a final answer.
Every tool execution is appended to an `actionLog` returned to the client and rendered in the
Activity Feed. Stream progress where possible.

## 6. Agent tools (function declarations)

Expose these as Gemini function declarations; implement each as a real server function:

- `list_tasks()` / `get_calendar_window(days)` — perception.
- `prioritize(tasks)` → risk-scored ordering.
- `decompose_task(taskId)` → ordered sub-steps + effort estimates.
- `find_free_slots(durationMin, byDate)` → reads Google Calendar (user token).
- `schedule_block(taskId, startISO, endISO, title)` → **creates a real Calendar event**.
- `draft_communication(kind, context)` → extension/reschedule email draft.
- `generate_artifact(taskType, context)` → outline / draft / prep questions.
- `set_smart_nudge(taskId, when, message)` → escalating, context-aware.
- `summarize_actions()` → human-readable activity trail.

**Demonstrate depth:** in a single rescue, chain **≥3 tools** (decompose → find_free_slots →
schedule_block → draft_communication). One-shot answers read as "chatbot." Handle failure
mid-loop (slot taken → re-plan) — recovery is peak agentic behavior.

## 8. Coding standards & guardrails

- **Keep it running.** After each change, ensure `npm run dev` works. Never leave the build broken.
- **The Agent Activity Feed is mandatory** and must update live as tools run.
- **Real error handling:** Gemini/API failures, empty calendar, denied permissions, no free slots.
  Fail gracefully with a user-visible message; never crash the page.
- **Secrets:** `GEMINI_API_KEY`, Firebase config, OAuth client ID in env (`.env.local`, never committed).
  Provide `.env.example`. Use Secret Manager for Cloud Run.
- **Commits:** small, logical, descriptive. Update `PROGRESS.md` (what's done / next / known issues)
  after each step.
- **Deployable always:** `output: 'standalone'`, a working `Dockerfile`, no node-only code paths that
  break on Cloud Run.
- **Demo-ready seed data:** include a "Load demo week" button so a stranger (judge) gets instant value.
